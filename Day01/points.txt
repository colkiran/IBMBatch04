Virtual Environment
--------------------
Virtual environment is a isolated space where you can work on your python projects.

to create a virtual environment first we should have the virtualenv lib installed in our machine

to check if the virtualenv module is present then we use a command 

  virtualenv --version


else we can install the virtualenv module

pip install virtualenv

Now we need to create the virtual environment

virtualenv path + virtualenv_name

Once created then we need to activate the virtual environment

path\Scripts\activate

(virtualenv_name) c:\users\jack>

we can now install all necessary libraries in the virtual environment. To check the installed libraries then we use a command 

pip list

lists all the modules installed on the current Virtual Environment

to deactivate the virtual environment

(virtualenv_name) c:\users\jack> deactivate

c:\users\jack>

your prompt will look as shown above

to delete the virtual environment

rm -rf virtual_env_name

The virtual environment folder will be deleted

doc strings
-----------
this is a place on top of the functions where we can write necessary information about the function like how many arguments the function takes and what are the data types of these functions

this information will be listed when we run help on functions

help(fun_name)

these doc strings are enclosed in "" or '' if it is a single line
else will be enclosed in """  """ or  '''  ''' for multiple lines

these strings will not be assigned to any variables

python interpreter will not touch the strings that are created
